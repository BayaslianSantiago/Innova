Bases de Datos

Unidad 3: Introducción a los Diccionarios y el formato Clave-Valor en Python.

El Problema

¿Qué pasa con las Listas?

La pérdida de significado.

En 1er Año vimos las Listas [] para guardar varios datos. Pero tienen un problema gigante: los datos solo se identifican por su posición.

# Lista de estadísticas
jugador = ["Messi", 10, "Delantero"]

# ¿Qué es el 10? ¿Goles? ¿Dorsal? ¿Partidos jugados? ¡No sabemos!
La Solución

Diccionarios {}

Estructura de Clave-Valor.

Los Diccionarios usan llaves {} y le ponen una etiqueta (Clave) a cada dato (Valor). Es como la ficha técnica de un jugador.

# Ahora el dato tiene nombre y apellido
jugador = {
  "nombre": "Lionel Messi",
  "dorsal": 10,
  "posicion": "Delantero"
}
Operaciones

Acceder y Modificar

Controlando el sistema de datos.

Para leer un dato o modificarlo, usamos los corchetes [] llamando a la etiqueta (Clave) que queremos buscar.

# 1. Leer un dato:
print("El capitán es: " + jugador["nombre"])

# 2. Modificar un dato existente:
jugador["dorsal"] = 30 # Se fue al PSG :(

# 3. Agregar un dato nuevo:
jugador["campeon_del_mundo"] = True
