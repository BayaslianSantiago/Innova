Planificación Anual: Ciencia de Datos y Análisis

Profesor Bayaslian Santiago

1. Fundamentación

Dado el tiempo respectivo de las clases (1 hora semanal), la materia se centrará en la Alfabetización y Visualización de Datos utilizando herramientas profesionales de la industria.

Aprovechando las Chromebooks institucionales, los alumnos de último año trabajarán con entornos de desarrollo en la nube (GitHub Codespaces / VS Code Web) en lugar de herramientas estáticas. Aprenderán a leer gráficos, entender métricas reales (KPIs) y construirán Tableros de Control interactivos y profesionales utilizando Python y la librería Streamlit. El objetivo es que se lleven habilidades de desarrollo moderno directamente aplicables a su futuro laboral.

2. Objetivos de Aprendizaje

  • Diferenciar entre datos crudos e información útil mediante el análisis de KPIs.
  • Manejar el control de versiones básico en la nube utilizando GitHub.
  • Utilizar entornos de desarrollo cloud (Codespaces / Visual Studio Code Web).
  • Desarrollar aplicaciones web interactivas orientadas a datos utilizando Streamlit en Python.
  • Utilizar IA de forma ética para acelerar la escritura de código y el análisis de datos.

3. Ejes Temáticos (Formato Cuatrimestral)

Primer Cuatrimestre: "Del Dato al Tablero de Control"

  • Unidad 1: El mundo de los Datos y KPIs. Dato vs Información. ¿Qué es un KPI? La regla SMART. Por qué los "Likes" suelen ser métricas vanidosas.
    • Evaluación: TP Unidad 1 (Definir 3 KPIs para un negocio ficticio).
  • Unidad 2: Fundamentos de Dashboards. ¿Qué es un Tablero de Control? Lectura visual rápida. Elegir el gráfico correcto según lo que se quiere mostrar.
    • Evaluación: TP Unidad 2 (Criticar un gráfico mal hecho y proponer uno mejor).
  • Unidad 3: Arquitectura de la Información. Cómo estructurar un Dashboard antes de programarlo. Wireframing y bocetado.
    • Evaluación: TP Unidad 3 (Diseñar la estructura lógica de un tablero utilizando post-its o herramientas de dibujo web).
  • Proyecto Integrador C1: Diseño de un Dashboard "Unplugged". Los alumnos deberán analizar el desempeño de un negocio ficticio diseñando un Tablero de Control en papel, Excel o PowerPoint, dejando la lógica e idea de interfaz preparadas para programar en el futuro.

Segundo Cuatrimestre: "Visualización Web con Streamlit"

  • Unidad 4: GitHub y Entornos Cloud. Qué es un repositorio. Creando nuestra primera cuenta de GitHub y explorando GitHub Codespaces. Primeros pasos con Pandas en la nube.
    • Evaluación: TP Unidad 4 (Crear un perfil de GitHub, abrir Codespaces y ejecutar un script leyendo un CSV con Pandas).
  • Unidad 5: Dashboards Web con Streamlit. Introducción a la librería Streamlit. Cómo transformar el script en una aplicación web interactiva (botones, sliders, gráficos).
    • Evaluación: TP Unidad 5 (Levantar una app web básica con Streamlit que muestre un gráfico simple en el navegador).
  • Unidad 6: Storytelling y Copilotos. Uso de extensiones de IA Generativa en VS Code Web para ayudar a programar. Sesgos en los datos. Cómo presentar el Dashboard final.
    • Evaluación: TP Unidad 6 (Armar el guion de presentación de su aplicación).
  • Proyecto Integrador C2: Presentación Oral ("Pitch"). Los alumnos presentarán un Dashboard interactivo real (programado con Streamlit y desplegado desde GitHub Codespaces) sobre un tema de su interés, argumentando una "decisión a tomar" frente a la clase.

4. Criterios de Evaluación

El año lectivo será evaluado íntegramente mediante entregas prácticas, exposiciones y trabajo en el aula:

  • Entregas de Unidad: 1 Trabajo Práctico corto por cada una de las 6 unidades (Entregando links a repositorios de GitHub cuando aplique).
  • Proyectos Integradores y Defensa Oral (Pitch): 1 Proyecto integrador en grupo por cuatrimestre. La evaluación del Proyecto Integrador final culmina obligatoriamente con una presentación oral ("Pitch") frente a la clase.
  • Participación Activa: Se evaluará de forma continua la predisposición, el compañerismo y la participación activa durante las clases (aportar ideas, debatir sobre ética y ayudar a otros grupos).
  • Uso Ético y Crítico de la IA: Se fomenta y espera el uso de herramientas de Inteligencia Artificial como asistentes de programación y análisis. Sin embargo, su uso debe estar acompañado de pensamiento crítico: el estudiante debe comprender, explicar y justificar cada concepto implementado por la IA, demostrando una apropiación real del contenido de la materia.
