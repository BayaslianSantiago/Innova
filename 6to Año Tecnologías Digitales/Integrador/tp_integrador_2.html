Proyecto: Full-Stack Dashboard

Ciencia de Datos y Análisis  ·  Trabajo Integrador Final (C2)

EVALUACIÓN ANUAL FINAL 6to Año

Objetivo: Construir un Dashboard web interactivo 100% funcional reuniendo los conocimientos del año entero. Van a tomar el boceto que diseñaron en el primer cuatrimestre, crear los datos reales, analizarlos con Pandas y publicarlos usando Streamlit.

Fase 1: Los Datos (Pandas)

  • La Base: Creen un archivo CSV inventado en Codespaces (mínimo 20 líneas) con datos que apliquen al negocio que eligieron en el primer cuatrimestre.
  • Lectura y Limpieza: En su app.py, importen pandas, lean el archivo y asegúrense de usar .dropna() por si hay celdas vacías.

Fase 2: La Web (Streamlit Top-Down)

Su código de Streamlit DEBE respetar obligatoriamente la regla de las 3 capas.

  • Capa 1 (KPIs): Usen st.columns() y st.metric() para mostrar arriba de todo los 3 o 4 indicadores clave que definieron hace meses.
  • Capa 2 (Tendencias): Usen las funciones de gráficos (ej: st.line_chart o st.bar_chart) pasándoles los datos agrupados con Pandas para contar el problema o tendencia de ventas.
  • Capa 3 (Detalles): En el fondo de la pantalla, incluyan un st.dataframe() para mostrar la tabla completa para que el jefe pueda investigar.
🚀

Bonus Track: Despliegue Público (Opcional)
Si quieren lucirse, vinculen su cuenta de GitHub con share.streamlit.io y publiquen el Dashboard gratis en internet para poder mandarle el link real a sus familias y ponerlo en su Curriculum.

Defensa Final

La evaluación es oral. Lanzarán su servidor (streamlit run app.py) frente al proyector del aula y demostrarán cómo su código de Python se transformó en una aplicación profesional, logrando contar una "Data Story" convincente.
